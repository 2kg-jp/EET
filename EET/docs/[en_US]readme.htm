Introduction

Baldur's Gate: Enhanced Edition Trilogy (EET) is a modification for Baldur's Gate II: Enhanced Edition that merges whole saga into one game and provides continuity between the events of Baldur's Gate, Tales of the Sword Coast, Shadows of Amn, Throne of Bhaal and Siege of Dragonspear expansion (also for both Black Pits adventures). The name of this this modification probably rings a bell to those who have been following IE modding scene, thanks to the classic mod that inspired this project, called Baldur's Gate Trilogy (BGT), developed by Bardez and Ascension64. While EET remains mostly faithful to the premise of the BGT, it shouldn't be considered a direct continuation of that project. In fact both mods share little to no common code and took different design decisions in many aspects (more information in F.A.Q).

Baldur's Gate: Enhanced Edition Trilogy has two main goals:

1. Uniting of the Baldur's Gate series into a single, continuous world that doesn't feel like 4 separate campaigns

  • Play all Infinity Engine Baldur's Gate saga games in Baldur's Gate II: Enhanced Edition engine. One game, complete package
  • Seamless experience, NPC continuity, better internal consistency between the games
  • United journal system that keeps track of all 24 chapters of your journey, from the beginning in Candlekeep up until the end of Throne of Bhaal expansion (this feature will be available after patch 1.4 for BG2:EE)
  • No restrictions in travelling between BG1, SoD, BG2 and ToB areas, once they are discovered (currently going back to BG1 areas is delayed to Chapter 7+, after patch 1.4 for BG2:EE this restriction will no longer exist). All areas accessible from the same worldmap

2. Becoming a mod friendly platform with additional modding possibilities that comes with fulfilling the first goal

  • Solution for mods designed to affect BG1 and BG2 content at the same time (preserved variables, character adjustments, world state, saves, things that can vary between installations like spell references assigned by WeiDU etc.)
  • A way to create new content designed for higher level parties in areas that normally wouldn't exist in game (vanilla games) or would be blocked after certain points of the story (original BGT)
  • Intuitive conventions for renamed BG:EE files and modders resources that can be used for adopting existing mods
  • Limited support for mods installed on BG:EE previous to installing EET on BG2:EE

Requirements

EET requires the following:

The BG2:EE installation (patch v1.3 is required) should be a "clean" installation. Translations and voice-overs are of course allowed but installing other (especially larger) mods that changes DLG and BCS files, before EET, may cause problems with later patching, so it's better to keep the installation as clean as possible.

The BG:EE installation (patch v1.3.2053 is required) should be either "clean" installation or with any combination of mods listed in the F.A.Q section.

Siege of Dragonspear expansion installed on BG:EE will be required after patch v1.4 for BG2:EE is released (currently EET is installable without it).

BG2:EE engine doesn't support BG:EE TIS files which results in quite inflated disk space requirement during installation (all BG:EE PVRZ TIS files needs to be extracted and modified). Currently at least 6.94 GB free space is needed during installation. For virtual memory purposes, at least 500MB-1GB+ (depending on your operating system) more free disk space is recommended during installation. Once the installation is complete EET modification occupies at least 3.68 GB.

If Beamdog will add support for BG:EE TIS files in BG2:EE engine above mentioned free disk space requirements will be much lower: at least 2.52 GB during installation, and only 1.92 GB total size of new data after installation.

Mods installed on BG:EE previous to installing EET on BG2:EE obviously increase free disk space requirements.

BP-BGT Worldmap is fully supported. Even though EET can work without it, some mods may require this special map (should be mentioned in the mod's readme, if this is the case). If you're going to install it, do so at the end, after any BP-BGT Worldmap compatible mods that add new areas.

Installation and Uninstallation

Installation

It is recommended that you disable virus scanners, spyware scanners, and other background on-the-fly file scanners during the installation. These programs have been known in some cases to interfere with the WeiDU installation process.

Installation within Program Files or Program Files (x86) is not recommended due to the potential for O/S related issues (e.g. UAC trouble). Unless you are sure that you have full UAC permissions consider moving BG:EE and BG2:EE outside this directory.

The installation checks that the Baldur's Gate: Enhanced Edition installation directory you typed contains a full game. If the installation needs to be aborted to correct your BG:EE installation, you can safely close the installation window or press Ctrl+C.

You may uninstall Baldur's Gate: Enhanced Edition if desired. If you think you may want to-reinstall EET at a later date, leave BG:EE installed.

You can either use Big World Setup to download, extract and install EET and other mods (well, not yet, but EET support will be added there soon) or install the mod manually:

  1. Extract EET-v#.7z to your Baldur's Gate II: Enhanced Edition install directory
  2. Run setup-EET.exe in your Baldur's Gate II: Enhanced Edition install directory and follow the installation prompts

Uninstallation and Reinstallation

Run setup-EET.exe in your Baldur's Gate II: Enhanced Edition install directory and follow the installation prompts

Auto-installation (for expert users)

Follow the first 2 steps mentioned above. Instead of directly running the executable go to the command line, navigate to your installation directory, and execute setup-EET.exe using the construction below. Alternatively, place the construction in a batch file or the Run... dialog in the Start menu.

setup-EET.exe --force-install 0 --skip-at-view --language X --args-list [flags] [path]

Parameters:

X

0 - English
1 - Polski (Polish)
2 - Deutsch (German)
[flags]

Any combination of below flags should be typed as one string (no whitespaces)

0 or 1 - suppress prompt for choosing the WeiDU TLK language directory if different language than selected for EET installation is specified within weidu.conf file located in BG:EE directory. 0 - leave the weidu.conf as it is, 1 - update weidu.conf to use the same language as EET installation.

m or M - suppress failed installation message that normally shows up if BG:EE installation contains mods not listed on EET BG:EE mods compatibility list (not recommended unless you really know what you are doing).

d or D - keep EET installation files (at least 3.2GB) after the installation is complete (for debug purposes only, they are not necessary to uninstall/reinstall the mod)

p or P - pre-define the full path to the Baldur's Gate: Enhanced Edition directory; if the path is invalid, setup will prompt for a valid path

q or Q followed by number from 0 to 9 (without whitespace) - allows you to change TIS conversion quality from default 0 up to 9. Not necessary as direct comparisons in zoomed images didn't present any quality loss of the TIS file. Values higher than 0 may slow down the installation process significantly

j or J followed by number from 0 to 16 (without whitespace) - another tile2ee tool setting. It allows to change default 0 value (auto-detect) up to 16. The number represents amount of "jobs" that this tool will perform at once during conversion (multi-tasking). It's recommended to keep the value at default - this way tool will automatically detect suggested amount of jobs depending on number of cores and threads of your CPU.
[path] the full path, in quotes, to your Baldur's Gate: Enhanced Edition installation directory, used in conjunction with p or P in [flags]

Example:

setup-EET.exe --force-install 0 --skip-at-view --language 0 --args-list 1q0j0p "C:\Program Files (x86)\BeamDog\Games\00766"

This construction installs EET with the English language, automatically updates weidu.conf file located in BG:EE directory to use the same language as the one chosen for EET installation (if such unusual situation occurs), and pre-defines the Baldur's Gate: Enhanced Edition directory as C:\Program Files (x86)\BeamDog\Games\00766. q0 and j0 sets amount of jobs/quality of TIS to default values, so won't change anything (usage showcased here just as an example in case you want to increase them).

How to start the game?

  • Baldur's Gate - first icon in main menu (choosing this option allows you to play whole saga from start to finish - BG1, TotSC, SoD, BG2:SoA, BG2:ToB)
  • Baldur's Gate II: Throne of Bhaal - second icon in main menu
  • Black Pits II: Gladiators of Thay - third icon in main menu
  • Black Pits - start Black Pits II and choose new dialogue response during the first conversation in tavern: "You can tell a ballad with more subtleties and nuances if you have experienced something first hand. Listen to my story." (after finishing Black Pits portion you will again end up in Black Pits 2 tavern, so you can continue this adventure with the same party.)
  • Baldur's Gate II: Shadows of Amn - start Baldur's Gate, equip a book (darker one) that has been added to your inventory and use it in game (your character will receive missing XP if it's below 95k and Bhaal powers. Mages will automatically receive memorized spells appropriate to BG2 starting level, in the same way as if you would choose "pick for me" during character generation)
  • Tutorial - start Baldur's Gate, equip a book (brighter one) that has been added to your inventory and use it in game

What are the differences between EET and the original Baldur's Gate: Enhanced Edition?

Nothing significant, at least in the way you're going to experience it. Technically there are some minor adjustments to make the BG1 portion of the game compatible with BG2 resources but it's not something that you will notice in game. The only gameplay difference is the way the Iron Crisis items shattering is handled. Unlike vanilla game, in EET Iron Crisis affects not only normal steel weapons (0.5% chance for each attack in EET, compared to 1% in BG1) but armours (0.2% on hit), shields (0.2%) and helmets (0.1%) too. The effect is true for your party and enemies. It can be considered more a restoration rather than tweak, as from game files it's pretty clear that at least armours and shields were meant to break since they have corresponding flags set and unused icons are present too. If you don't like this feature then install EET Tweaks components that will allow you to make it work exactly as in the vanilla game. Other than this obviously the game won't finish after the BG1 end movie. Whole game is also using a new Worldmap.

What are the differences between EET and the original Siege of Dragonspear expansion?

It's a little too early to write something for certain, as the compatibility for Siege of Dragonspear is not implemented yet. For now EET aims to include this expansion seamlessly which means you won't be restricted to travel to areas from the main game once you start the expansion. The history itself will be continuous so instead of starting from chapter 1, Siege of Dragonspear will take place over the chapters 8-14. SoD worldmap is going to be merged with EET Worldmap.

What are the differences between EET and the original Baldur's Gate II: Enhanced Edition?

For the most part everything remains the same. There are however few new features:

  • Full seamless transition between BG1 and BG2 portion of the game for join-able NPCs present in both games, regardless if you have them in party during transition (in vanilla game only CHARNAME could migrate to BG2 via character or save import functionality).
  • Full transition between BG2 and ToB portion of the game for join-able NPCs (in vanilla game only those in your party were moved to ToB - other characters were spawned with new CRE files loosing all adjustments you've made to them at some point).
  • A way to go back to BG1 areas in BG2: Shadows of Amn (currently delayed to chapter 7 or later but after patch 1.4 for BG2:EE it will be possible to travel back as soon as you leave the Irenicus Dungeon).
  • A way to go back to BG1 and BG2 areas during Throne of Bhaal expansion (as soon as you leave Saradush).

What are the differences between EET and Baldur's Gate Trilogy (BGT)?

As mentioned in the beginning, technically EET is not a port of BGT for Enhanced Edition engine. Without going too much into details some noticeable (for BGT players) changes have been listed below. Things unique to Enhanced Edition content have been skipped, as it's kinda obvious that you won't find in BGT anything that has been created by Beamdog, such as Black Pits I and II campaigns, new NPCs and areas. BGT descriptions in the below table have been written mostly using Ascension64 quotes from the old topic, with the BGT/TuTu merger discussion.

Unhide/Collapse details

    BGT EET
    BG1 resource naming and conventions BGT has a radical system of renaming the BG1 areas. The area prefix is still "AR", but the numbers have no bearing on the original BG1 area numbers. This is because BG1 area numbers conflict with those in BG2, and maintaining 'compatibility', the numbers need to changed. As an example, Candlekeep is AR2600 in BG1 but AR0015 in BGT. For another, AR4400 in BG1 is AR3300 in BGT. Area .BCS, .BMP, .MOS, .TIS, and .WED files follow the same naming convention as the .ARE files.

    For other resources, BGT keeps the original BG1 resource name as strictly as possible. However, the progressive nature of BGT takes into account incompatibilities with modifications founded and completed before the BGT-WeiDU revolution. For example, the modification 'Never Ending Journey' uses IMOEN6.CRE, and so BGT compensates in this specific, case by renaming IMOEN6.CRE to IMOEN61.CRE. This is an unusual case, and only a few others are present. Typically, BGT will prefix with "BG". For example, other 'Tortured Souls' incompatibility, is WOLF.CRE, which BGT renames to BGWOLF.CRE. Due to 2 characters prefix there are instances where filenames had to be changed to fit in 8 characters limit (no common convention, usually by dropping 1 or 2 letters, usually at the end but not always).

    Conflicting spells from BG1 uses a little different naming convention - instead of prefixing them with "BG", these letters replaces first 2 letters of the spell (so SPWI becomes BGWI etc). The reason behind this is limitation of the engine which expects maximum 7 character spell names for some spell effects.

    BGT makes some technical changes to how the BG1 portion of the game operates within BG2:
    1. NPCs joinable in BG1 that also exists in BG2 (Edwin, Imoen, Jaheira, Minsc, Viconia) shares the same JOIN, POST and BANTER dialogue files. BGT appends BG1 dialogues at the end of BG2 files with additional variables to prevent incorrect text to show up (also forces mods to do the same).
    2. BGT for some reason changes all BG1 entrances and door names instead of living them as they were in BG1, which results in area incompatibility between platforms
    3. to make BGT compatible with mega-mods Ascension64 decided to make some unexpected changes that doesn't follow usual convention used for other files (Dynaheir BANTER file renamed from BDYNAH to BDYNA)     		EET has a straightforward system of renaming the BG1 areas. The numbers are untouched, but the prefix "AR" is changed to "BG". This maintains the consistency and validity of BG1 area references from a number of public documents, including walkthroughs, travel maps, and the like. Therefore, these documents can be referred to for an area number guide to EET, except that the prefix change from "AR" to "BG" must be considered when, for example, using the CLUAConsole, or when developing BG1 modifications. Area related files, all use the same naming guidelines as the .ARE files.

    Like BGT, EET keeps the original BG1 non area resource name as strictly as possible. There is no point in changing the name if both BG:EE and BG2:EE share the same resources. For conflicting resources BG1 ones have suffix with the underscore (_) to make them unique. Since the engine only supports 8.3 filenames, 8-character resources from BG1 have "0" character removed if replacing the last character would result in overlapping resource names - only 2 cre files: skelwa02.cre => skelwa2_.cre, skelwa03.cre => skelwa3_.cre, also used for random item tables (example: RNDWEP01 => RNDWEP1_).

    For conflicting spells from BG:EE "BG" letters replaces first 2 letters of the spell, like in BGT.

    On technical level EET aims to stay as close as possible to BG:EE:
    1. BG:EE JOIN (file that has all NPC normal dialogues) and POST (conversation when NPC is leaving the party) dialogue files for joinable NPCs uses standard EET renaming convention (underscore suffix) to prevent conflicts with BG2 files. This approach allows modders to write/edit NPC dialogues without worrying about conflicts between games and eliminates a problem with finding correct dialogue block numbers within the file. Only BANTER dialogues are appended (engine limitation).
    2. Entrances and Door names unchanged
    3. EET makes changes to BG:EE files only to prevent conflicts with BG2:EE
    BG1->SoA transition The BGT transition is a compulsory core component, and the nature of it can be considered the minimal-extended type. After killing Sarevok, the final movie displays but the credits do not show and force the player back to the main menu. Instead, the game continues and a string is displayed suggesting the player should go to the Duchal Palace. There, they will meet Belt. Interestingly, if Belt was killed during Sarevok's foible coronation, Belt still appears for the transition. Belt conjures up a cover story about continuing tensions between Baldur's Gate and Amn and asks the player to deliver a message to the Council of Six. The details are sketchy, but Belt teleports the players to an undisclosed location near Amn. There, a cutscene is only shown, where Shadow Thieves led by Mae'Var ambush the players and take them to Irenicus' dungeon. This cutscene also serves as the vehicle through which many journal entries are removed, items are checked for importation, and NPC transitions are made, if applicable. When in Irenicus' dungeon, the Shadows of Amn introduction movie is played, then Shadows of Amn proper commences.
    		 There's nothing that can be written here from the story standpoint. For now EET transition to BG2 starts right after the final BG1 battle. In future, this part of the game will be expanded to fully merge in upcoming Siege of Dragonspear official expansion, made by Beamdog and approved by Wizards of the Coast, so hopefully a quality content that will expand upon "leaving Baldur's Gate under circumstances much darker than anyone would have believed" quote from BG2 intro. As for the technical differences, EET transition script is not limited to the party members like in BGT. For example if you had Jaheira and Khalid any time during your game, and then dumped them some way down the road, Jaheira will still transition to BG2 portion of the game (missing XP from the BG2 version counterpart will be added, so you don't need to worry about it). If NPC dies then you won't find him/her in BG2 (except Imoen and Edwin - these 2 are required due to the plot). More information about it later in FAQ.
    BG2->ToB transition
    		 Not existing in BGT, uses vanilla system, which means NPCs summoned by Spirit in AR4500 will be newly created creatures (they will lose their original items, level, stats etc. even if you had them in BG2, but not in final BG2 confrontation).
    		 EET provides full transition for all vanilla NPCs that you had in party at some point. If you didn't hire NPC at any time during BG2 portion of the game you won't be able to even summon them (you don't know them, so they are not on the summon list). Same goes for dead former party members. BG1 NPCs that weren't present in BG2 will be mentioned, even if not summonable. More information about it later in FAQ.
    Journal and chapters system
    		 BG1 chapters have different numbers than in vanilla BG1 (Prologue is considered as Chapter 1 instead of 0, so chapter globals are incremented by one relative to the actual chapter number).

    During transition to BG2 chapters are reset back to 1 and all quest journal entries are wiped out (you start with a clean journal for BG2 portion of the game)
    		 EET uses the same chapter numbers as BG1/BG:EE for BG1 content (chapter from 0 to 7).

    Currently once BG2 starts the journal is cleaned out like in BGT, but after patch 1.4 for BG2:EE this behaviour will be changed to append chapters on top of each other, so you will end up with a continuous journal that register the story of your character during whole game (24 chapters in total)
    Conversion method
    		 BGT comes with all BG1 .ARE, .BCS, .DLG and many .CRE files. Files extracted directly from the BG1 installation directory are mostly graphic and sound/music resources. This approach was necessary due to amount of bugs present in the original BG1 that BGT aimes to fix. Resources that are not used in game, or being present in BG2 are not extracted.

    Thanks to this approach the installation is fast and fluid. According to BGT readme file installation time will depend on the abilities of your system, but as a rough estimate installation may take anywhere from 20 minutes to 1 hour. In my experience these estimates are rather pessimistic. You can expect the installation on mid range laptop to take closer to 10-15 minutes.
    		 EET took entirely different conversion method than BGT. Instead of coming with pre-made files, all resources are converted "on the fly", thanks to the great macros created by A64 in his Platform Conversion Utility (PCU), that have been expanded to cover whole game. This approach allows EET to be compatible with changing EE games content (massive differences in each patch released by Beamdog) with little to no changes in EET code. Every resource or reference to resource that need to be renamed, are identified during installation and updated if necessary. Additional benefit of this type of conversion is compatibility with many BG:EE mods (installed on BG:EE previous to installing EET on BG2:EE) without need for modders to adopt code to another platform.

    The downside of this approach is longer installation time compared to BGT. On mid range laptop (without SSD) it can take 30-40 minutes, and with dialogue heavy mods installed on BG:EE the installation may be even longer. Due to problem with watter that currently can be fixed only by extracting PVRZ files to TIS, you can add another 10-20 minutes to the installation time, depending on your CPU (this step will be skipped in future if Beamdog decide to add proper BG:EE TIS water support to BG2:EE engine at some point)
    Bug-fixes One of the main features of BGT is presenting BG1 story without bugs. Massive amount of work has been done to achieve the great BG1 experience.
    		 Beamdog still supports both BG:EE and BG2:EE with patches that aim to fix problems with the game. At this point EET is not actively going after vanilla game bugs. There are few minor fixes implemented for vanilla BG:EE content (they will be removed if Beamdog officially fix these problems).
    Travel back to areas
    		 Not possible in BGT after you reach the BG1->BG2 transition point.
    		 Currently EET allows you to travel back to BG1 areas when you reach chapter 7 in BG2. After patch 1.4 for BG2:EE this limitation will be removed, so you will be able to travel back even in the beginning of BG2. During ToB expansion travelling to BG1 and BG2 areas is possible right after leaving Saradush.
    Other
    		 1. BG2 engine.
    2. Weapons shattering as in BG1. Similar functionality to EET can be achieved via BGT Tweaks mod, although in more demanding and not mod friendly way (via expanding local scripts attached to all creatures in game).
    3. Familiars as in BG2, vanilla hardcoded implementation (same ones in both BG1 and BG2).
    4. BG2 portion of the game can be started from the main menu (SoA button replaces tutorial).
    5. Tutorial not available.
    6. Built in support for mega-mods (TDD, SOS, TS, CtB, RoT, NEJ).
    7. Built in worldmap that covers vanilla BG1 (map scaled down), TotSC and BG2 areas. ToB as a separate map (vanilla one). Optional support for BP-BGT Worldmap.
    8. BGT Tweaks.
    		 1. BG2:EE engine (enhanced modding possibilities like unhardcoded animation support, INI spawns, unlimited random loot and much more.
    2. Altered shattering system to make it mod friendly, more efficient performance wise, expanded to cover additional items types and affect everyone on the Swoard Coast during BG1 portion of the game.
    3. Familiars in BG1 portion of the game uses files from BG:EE, which are far weaker than their equivalents that can be summoned once you reach BG2 and ToB.
    4. In order to start BG2 portion of the game without completing BG1 you need to start a new BG1 game and do as has been explained here (engine limitation).
    5. As above - tutorial can be optionally started via one of the books that you will receive at the start of Candlekeep prologue.
    6. Built in support for BG:EE mods. Support for mega-mods will be added if needed once such mods become available for BG2:EE (although not via renaming conflicting vanilla BG:EE files as BGT does. Creating more headaches for modders doesn't sound like a good idea when the same outcome can be achieved via editing the mods itself).
    7. All games and expansions (BG1, TotSC, SoD, BG2, ToB) takes place at the same worldmap (vanilla map sizes combined into 1 worldmap in BG2 style). Optional support for BP-BGT Worldmap and new external worldmaps (engine feature).
    8. EET Tweaks.

How to add EET compatibility for other mods?

Adopting existing mods to conventions and technical features used by EET is relatively easy and intuitive thanks to modder's resources and tools included in EET package that can be used to drastically reduce time needed for such conversion. Please refer to the EET developer's documentation for more information.

What about compatibility with saves?

Old save games from original BG:EE and BG2:EE are not directly supported - you need to start a new game. You can import a character or use "save import" feature though. As for the save compatibility with different installations of EET, before removing current installation you should consider coping old dialog.tlk (also dialogF.tlk in some languages) to appropriate lang subdirectory in your new installation folder. Usually this step is enough to make your saves fully functional, even if you change installation order or use slightly different mod selection.

How does NPC continuity work in EET?

Character migration to BG2 currently starts right after killing Sarevok, but in future transition will start after finishing SoD expansion. Some characters are not continuous (those without BG2 content) and are destroyed immediately, whereas others appear in Shadows of Amn and so continuity is required. Continuity only occurs when a joinable NPC was in your party at any point during BG1 (even if you didn’t had that character in your end-game party he/she is still valid for transition) and is not dead. After character is moved to appropriate area some adjustments are being made if applicable (attribute changes, alignment, portrait, soundset, item, XP, dialogue and script updates). Otherwise, the appropriate level of the Shadows of Amn-resident joinable NPC is created or character doesn’t show up at all if NPC has been killed during BG1 (except Imoen and Edwin due to plot requirements).

Like in vanilla game at the end of BG2 your party migrates to ToB. What has been changed is Fate Spirit old party member summoning mechanics. It no longer spawns new versions of creatures but moves characters from BG2 areas to Pocket Plane. Fate Spirit summoning NPC list is created dynamically depending on who you met during your BG1 and BG2 travels. For continuity sake NPCs that you don’t know won’t even show up on the list. During Fate Spirit dialogue you can receive one of 3 responses to your request:

  • I cannot do as you ask, child of Bhaal. The one you seek is dead and thus forever beyond my reach. This thread no longer connects with yours. – originally used only for Yoshimo, now valid for every NPC that has been killed and left in that state.
  • I cannot do as you ask, child of Bhaal. The thread of the one you speak is beyond my sight, as it no longer touches yours. – this unused line in vanilla game has been restored to cover BG1 NPCs that don’t have content created in BG2. If you had a character in party at any time during BG1 (true for all NPCs, not only those with BG2 content) but that NPC didn’t join you at least once during BG2 than he/she will show up on the summoning list, but you won’t be able to summon such character.
  • As you wish. – NPC will be summoned. Spawned NPC is the exact same creature that you left in BG2 (items and other adjustments are preserved, dialogue and script updates are made, if applicable, XP is increased to default starting ToB XP if NPC has less experience).

What's the point in lifting restriction for travelling back to old areas?

Maybe it's just me (K4thos), but I've never accepted BGT as a true seamless experience because the game still felt to me like 3 completely separate campaigns with BG1->BG2 transition added on top of it. As you can read in the beginning of the readme, the main goal of EET is not simple merging of the games, but actually make it feel like a one, continuous journey. In order to achieve it EET lifts travelling to old areas limitations, provides continuous journal, uses the same worldmap during whole game, and offers seamless transitions between story arcs. The second goal of EET is becoming a mod friendly platform with additional modding possibilities that comes with fulfilling the first goal. Currently there is nothing interesting in old areas but we hope that modders will be interested in creating new adventures and surprises designed for higher level parties during BG2 and/or ToB portion of the game. Also thanks to the enhancements made to EE engine it's now possible to create custom spawns for high level parties without tons of code or conflicts, which may prove useful for hypothetical mods that could try to make old places interesting for returning parties. In other words it can be treated as a playground for modders to add new stuff to the game.

Why EET aims to open access to BG1 areas so early in BG2? It can unbalance the game!

Once the patch 1.4 for BG2:EE hits, EET will by default remove the current delay for visiting old areas (due to technical matters it is currently delayed to chapter 7+). Free access to old areas indeed can unbalance early BG2 game, as it will be easier to get good items that the party could store somewhere during BG1 portion of the game, which creates another way to exploit the game for players that cannot refrain themselves from taking advantage of it. For this reason, once the functionality is fully implemented you will be able to install new EET Tweaks component that can block access to them entirely or delay it for the chapter of your choice. The reason why such tweak is an opt-out option (you need to install separate component to get rid of it) instead of opt-in option (the other way around) is mostly related to the above mentioned second goal of the EET. Optional open travelling by default would drastically reduce amount of people playing with it (that's always the case for optional stuff) which is an equivalent of smaller player base for mods that aims to add new content to BG1 areas in early/mid BG2 portion of the game. So the end result would be smaller chance that anyone would be even interested in creating such content. Personally we would love to see stuff like for example an option to ask Grand Dukes of the Baldur's Gate city for help with gaining money to fee Imoen and other stuff that makes sense lore wise. If someone will take advantage of these new possibilities remains to be seen. Leaving the base game without such limitations will hopefully positively contribute to achieve this goal.

Can I play EET in cooperative multiplayer?

Yes. EET supports up to 6 people in cooperative, cross-platform multiplayer from start to finish. Let us know if you notice any problems.

Troubleshooting

Fatal Installation Errors

"(...) BG:EE mod is not recognized by EET, so it can't be installed on BG2:EE"

Due to reasons specified in FAQ, EET limits "on the fly" BG:EE resources conversion to vanilla BG:EE installation and mods listed in the compatibility list. Adding other mods is fast and easy, so the list will grow over time.

"TLK for language: (...) not found in BG:EE directory"

Check the selected BG:EE language directory if the TLK file is indeed there

I have Baldur's Gate: Enhanced Edition installed, but the installation does not accept the directory that I type in.

The most likely scenario is that your installation is broken. Try reinstalling Baldur's Gate: Enhanced Edition. Also you can try parsing the BG:EE directory using command line as described in the installation section.

WeiDU stopped the installation because of error.

You may not have a clean installation of Baldur's Gate II: Enhanced Edition or some resources are corrupted

  • When the installation prompts for install again, quit the installation
  • Try reinstalling Baldur's Gate: Enhanced Edition and/or Baldur's Gate II: Enhanced Edition
  • If you have other modifications installed, try uninstalling those and then install EET
  • If you still experience problems, post the SETUP-EET.DEBUG file, located in your Baldur's Gate II: Enhanced Edition installation directory, in the mod forum

Minor Installation Errors

The mod finished installing successfully, but the message is "Installed with Warnings".

There are several additional precautions added into EET installation code to inform the player if something went wrong. Open up SETUP-EET.DEBUG located in your Baldur's Gate II: Enhanced Edition installation directory and search (ctrl+F) for the word "warning". If you can't figure out how to fix the problem post this debug file in the mod forum. Playing with installation that showed warning is not recommended.

During the installation, I constantly get errors saying "There is no disk in your drive".

This occurs if you did not follow "Requirements" section of the readme file.

  • If you have not installed yet, or as you read this the installation is halted directly after you were asked to enter your Baldur's Gate: Enhanced Edition directory with the disk check, abort the installation immediately.
  • As you read this, if the installation is halted any time after you were asked to enter your Baldur's Gate: Enhanced Edition directory, press 'Cancel' to every single prompt that occurs, and allow the installation to finish (or fail to finish, just allow WeiDU to stop it itself). If the installation was able to finish, uninstall EET.
  • Follow the installation procedures correctly without omitting "Requirements" section.

In-game problems

The game has noticeable lag in Candlekeep and other open areas.

If your computer meet or exceed the system requirements for BG2:EE than this problem may be related to uncompressed TIS files currently used by EET. It has been reported that some integrated Intel graphic cards (HD3000 for example) need quite recent GPU drivers to work correctly with uncompressed TIS files. The problem will fix itself right after you install latest drivers.

Corners in interior areas are not displayed correctly.

This is a known BG2:EE engine bug that will be fixed by Beamdog in next patch (1.4).
